## Design Context

### Users
Primary user is Néstor De León, a teacher, researcher, writer, and developer who manages a living archive of academic, cultural, educational, and software projects. Secondary users are collaborators or future-you loading a local snapshot to inspect relationships, sources, and project status.

The main job is to import or sync a snapshot, navigate the project graph, inspect the central profile, and drill into connections without needing a remote database. The interface should support both Spanish and English and keep local data handling obvious and trustworthy.

### Brand Personality
The brand feels editorial, archival, intelligent, warm, and grounded. It should communicate order without coldness and depth without clutter.

Three-word personality: archival, human, rigorous.

The emotional goal is confidence. Users should feel they are working inside a curated memory system, not a generic dashboard. Small moments can add calm and delight, but clarity and trust come first.

### Aesthetic Direction
Current visual cues point to a warm paper-based palette with terracotta, slate, brass, and ink tones; serif headlines; mono metadata; and a restrained glass/brutalist hybrid for cards, nodes, and controls. The overall effect is more editorial workspace than corporate SaaS.

Typography is already strongly defined: Fraunces for expressive display moments, Instrument Sans for body and controls, and IBM Plex Mono for technical labels and system metadata.

The UI should keep the graph as the centerpiece, use visible hierarchy for profile, project, and connection states, and preserve the local-first language around import, restore, sync, and visibility.

Anti-reference: glossy generic dashboards, over-rounded SaaS UI, neon-heavy palettes, or anything that hides the sense of a curated working archive.

### Design Principles
1. Make the graph the primary navigation model and keep everything else supportive.
2. Preserve Spanish-first clarity in system copy, with English as a parallel mode rather than a second-class fallback.
3. Use warm, tactile, archival visuals to reinforce trust and authorship.
4. Show provenance, status, and visibility clearly so users always know what is local, published, active, or exploratory.
5. Keep the interface responsive and lightweight so it works well for both desktop exploration and narrower screens.
